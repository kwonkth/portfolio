Survey Studio – 설문 생성·응답·분석 플랫폼
2025.09 – 2024.11 · 팀 프로젝트 (팀명: 세얼간이 · 권태형 · 하경완 · 함지원)

설문 생성부터 응답 수집, 통계 분석까지 한 번에 처리할 수 있는 웹 기반 설문 플랫폼입니다. Cloudflare Pages + Pages Functions + D1 조합으로 프론트엔드·백엔드·데이터베이스를 단일 환경에서 운영하도록 설계했습니다.

Frontend

HTML CSS Vanilla JavaScript Chart.js
  • 순수 HTML/CSS/JS 기반으로 설문 생성 · 응답 · 대시보드 · 통계 화면 구현
  • 공통 컴포넌트(카드, 버튼, 모달)를 재사용해 일관된 UI 제공
  • Drag & Drop, 통계 차트 등 인터랙션을 모두 Vanilla JS와 Chart.js로 직접 구현

레포지토리 · github.com/kwonkth/survey-project

Backend (Cloudflare Pages Functions)

Cloudflare Pages Functions JavaScript (ES Modules) REST API
  • Cloudflare Pages Functions(Workers 런타임) 위에서 JavaScript ES Modules로 백엔드 구현
  • onRequest* 패턴으로 설문, 응답, 폴더, AI 설문 생성 등 REST API 엔드포인트 설계
  • 프론트엔드와 fetch 기반으로 통신해 설문 CRUD, 응답 수집, 통계 데이터를 주고받음

Database (Cloudflare D1)

  • wrangler.toml에서 D1 바인딩 이름을 surveyforge로 지정
  • env.surveyforge.prepare(...).run()/all()/bind() 형태로 SQL 직접 실행
  • surveys: 설문 메타데이터(제목, 설명, JSON 구조, status, folder_id, created_at, updated_at)
  • results: 설문 응답 데이터 및 통계 계산용 원본 저장
  • folders: 설문 폴더링/분류 기능용 테이블
  • 대부분의 핸들러에서 CREATE TABLE IF NOT EXISTS로 테이블 자동 생성 로직 포함

AI / 외부 API

  • 파일: functions/api/generate-survey.js
  • 모델: anthropic/claude-haiku-4.5 (OpenRouter Chat Completions)
  • OPENROUTER_API_KEY 환경 변수를 사용해 서버 측에서 fetch로 직접 호출
  • 응답 JSON을 설문 문항 구조에 맞게 정규화하여 프론트엔드에서 바로 사용할 수 있는 형태로 반환
  • /styles/*.json에 설문 스타일 템플릿을 정의하고, AI 프롬프트에서 불러와 테마/스타일이 적용된 설문 초안 생성

문제 해결 과정

  • 문제
    설문을 "재미있게 만들기"라는 요소가 매우 주관적이라, 수작업으로는 일관된 퀄리티의 설문을 만들기 어려웠습니다.
    이를 자동화하기 위해 AI 기반 설문 생성 기능을 붙였지만, 설문 스타일·문항 구조를 JSON 템플릿으로 관리하는 구조상 일부 모델(Gemini)의 JSON 출력이 깨지거나 형식이 달라지는 문제가 반복되었습니다.
  • 접근
    /styles/*.json 템플릿을 LLM에 함께 전달하는 구조는 유지하되, "정확한 JSON을 안정적으로 반환하는 모델"을 찾는 데 집중했습니다.
    OpenRouter API를 통해 여러 모델을 비교 테스트한 결과, Gemini 계열은 응답 형식이 일정하지 않아 파싱 오류가 잦았던 반면,
    Claude Haiku 4.5는 긴 문항 구조와 스타일 정보를 포함한 프롬프트에도 구조화된 JSON을 안정적으로 반환한다는 점을 확인했습니다.
  • 프롬프트 · 검증 개선
    설문 스키마를 명확히 명세하고, "반드시 JSON만 출력"하도록 조건을 강화했습니다.
    또한 서버 측에서 응답을 파싱할 때 형식이 조금이라도 어긋나면 자동 수정 또는 오류를 반환하는 검증 로직을 추가해, 모델 교체 후에도 안정성을 확보했습니다.
  • 결과
    JSON 템플릿 기반 자동 설문 생성 기능이 안정적으로 동작하게 되었고, 생성되는 설문의 완성도와 일관성이 크게 향상되었습니다.
    사용자는 "어떤 유형의 설문을 만들고 싶은지"만 선택하면, 바로 배포 가능한 수준의 설문 초안을 얻을 수 있는 흐름을 구축했습니다.
    기술적 제약(Gemini의 JSON 호환 문제)을 우회하고, 가장 적합한 모델(Claude Haiku 4.5)을 선택해 기능의 품질을 실질적으로 끌어올린 사례로 정리할 수 있습니다.

MealMate – 식단 추천 & 공유 서비스
2023.08 · 24시간 해커톤 (3인)

사용자의 선호도와 알레르기 정보를 기반으로 식단을 추천하고, 팀·동아리 단위로 식단을 공유할 수 있는 모바일 웹 앱입니다.

역할 (개인 기여)

  • Figma로 핵심 플로우(온보딩 → 설문 → 추천) 와이어프레임 작성
  • 모바일 퍼스트 UI 개발 및 반응형 레이아웃 구현

문제 해결 사례

  • 문제 · 해커톤 제한 시간 내에 전체 기능 개발이 어려운 상황.
  • 접근 · Notion으로 MoSCoW(Must/Should/Could/Wont) 우선순위를 정리하고, 핵심 가치(“나와 팀이 오늘 무엇을 먹을지 빠르게 결정”)에 직접적으로 기여하지 않는 기능은 과감히 제외했습니다.
  • 결과 · 핵심 플로우에만 집중해 심사 데모를 안정적으로 마쳤고, “제약 조건 속에서 우선순위를 잘 정한 팀”이라는 평가를 받았습니다.
